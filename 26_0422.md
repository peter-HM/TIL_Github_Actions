# Health check

- deploy.yml에서 health check step을 추가해 엔드포인트로 확인할 수 있다.

다음은 사용한 코드 예시

```

name: Health check
uses: appleboy/ssh-action@v1
  with:
    host: ${{ secret.EC2_HOST }}
    username: ${{ secret.EC2_USERNAME }}
    key: ${{ secret.EC2_SSH_KEY }}
    script: |
      sleep 10
      response= $(curl -s -o /dev/null -w "%{http_code}" http://localhost:8000)
      if [ "$response" = "200" ]; then
        echo " ✅ Health Check 성공: $response"
      else
        echo "❌ Health Check 실패: $response"
        exit 1
      fi

```

- sleep 10 : 컨테이너가 뜰 떄까지 10초 대기
- curl: 앱에 요청을 보내기
- -s : 조용한 모드 (진행바 숨김)
- -o /dev/null : 응답 내용 버리기
- -w "%{http_code}" : 상태 코드만 가져오기

해당 로그 확인은 actions로 확인이 가능

# Estimate Downtime

배포 중 서비스가 얼마나 중단되는지 측정하는 것

local 환경에서 다음 코드를 통해 요청을 보내 시간을 측정

```

while true; do
  response=$(curl -s -o /dev/null -w "%{http_code}" http://52.79.204.150:8000)
  echo "$(date '+%H:%M:%S') - $response"
  sleep 1
done
```

Downtime이 약 3초 발생. 000 = 서버에 아예 연결이 안됨.

- docker compose down --> docker compose up 까지.

즉, down을 하고 dockerhub에서 pull하는 period 3sec가 downtime을 유발. --> pull을 먼저 하여 image down 받는 시간을 줄일 수 있다.
