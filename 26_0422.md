# Health check

- deploy.yml에서 health check step을 추가해 엔드포인트로 확인할 수 있다.

다음은 사용한 코드 예시

```

name: Health check
uses: appleboy/ssh-action@v1
  with:
    host: ${{ secret.EC2_HOST }}
    username: ${{ secret.EC2_USERNAME }}
    key: ${{ secret.EC2_SSH_KEY }}
    script: |
      sleep 10
      response= $(curl -s -o /dev/null -w "%{http_code}" http://localhost:8000)
      if [ "$response" = "200" ]; then
        echo " ✅ Health Check 성공: $response"
      else
        echo "❌ Health Check 실패: $response"
        exit 1
      fi

```

- sleep 10 : 컨테이너가 뜰 떄까지 10초 대기
- curl: 앱에 요청을 보내기
- -s : 조용한 모드 (진행바 숨김)
- -o /dev/null : 응답 내용 버리기
- -w "%{http_code}" : 상태 코드만 가져오기

해당 로그 확인은 actions로 확인이 가능

# Terraform


## Traditional Deployment

- Business needs new product

- Solution architect / Tech lead design the plan the hardware

- Procurement team (financial team) purchased or leases physical servers, storage devices, and networking equipment based on their needs

- Infrastructure work

On-prem IT staff / DC engineer install the hardware

System / Network admin do BIOS, RAID configurations, and network settings

DevOps, SRE install environment configuration

- Deploy application by SWE

### Cons

- slow deployment

- Expensive cost upfront

- Automation limit

- High human error

- Inconsistency of utilization(peak / off-peak hours)

## Virualization / Cloud

- Efficient for deployment

- However it is hard to manage in large organization

- Companies try to solve those problem to create in-house application in Shell, Python, Ruby, Perl or Powershell.

### Infrastructure as Code

- Configuration management : ANSIBLE, puppet

Install & management software

Version control

Idempotent configuration ( 항상 똑같은 결과를 도출한다.)

Maintain the basic infrastructure

- Server Templating : Packer, docker, GCP, 

Bootstrap software for the environment

VM or Docker image

Immutable infrastructure (변하지 않는다.)

- Provisiong Tools : Terraform, AWS ClouudFormation, Pulumi(python)

Deploy immutable infra resources (Server, DB, Network and etc)

Support various providers

## What is terraform?

- Terraform is an open-source infra as Code tool (IaC)

- It enables users to provision, manage and maintain cloud infra in a safe, predictable, and efficient manner/

- It allows users to define and provision infra using a declarative configuration language HCL (HashiCorp Configuration Language) or Json


## How it works?

- Terraform utilize a concept called "provider" to interact with various cloud providers (e.g. AWS, Google Cloud, Azure) and other services (e.g. Github, K8s)

- Steps

a) Users writhe configuration files specifying the desired infrastructure components

b) Terraform builds an execution plan detailing how it will create, modify, or delete resources to achieve the desired state. (Modulization을 하게 된다. example. S3라고 한다면 마치 class를 선언하고 instance로 불러 오듯이 미리 설정을 한다면, 새로운 S3가 오면 바로 적용될 수 있게 한다.)

c) After reviewing the plan, users can execute it uing the terraform apply command, and terraform will provision the infrastructure accordingly.

## Extra Benefits

- Terraform also supports state management, enabling users to track changes in infra and share state files with teammate.

- This helps ensure consistency and reliability when managing large-scale, multi-cloud infrastructure.
