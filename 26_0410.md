on : or의 개념이다. 즉, 하나만 만족해도 가능하다. 만약 AND 조건이 필요하다면, job안에 별도의 로직으로 처리해야 한다.

```
name: My First Workflow
    on: workflow_dispatch
    jobs:
        job-1:
            runs-on: ubuntu-24.04
            steps:
                - run: echo "

```

job 간의 변수 전달

job은 완전히 독립된 컴퓨터에서 실행되기에 복잡한 면이 존재.

producer > 컴퓨터 A
vaildator > 컴퓨터 B
reporter > 컴퓨터 C
--> 그렇기에 변수를 전달하기 위해선 github시스템을 택배처럼 이용해야만 한다.

4 Steps

1) step에서 값 생성 및 저정
steps:
    - id: set-message
    - run: echo "message=workflow-complete" >> "$GITHUB_OUTPUT"

# key=value 형식으로 저장해주어야 한다.
# $GITHUB_OUTPUT은 임시 보관함

2) job의 output에 등록

outputs:
    messgae: ${{ steps.set-message.outputs.message }}

3) Github 시스템이 값을 보관

producer job 완전히 종료 > github 시스템이 outputs.message = "workflow-complete" 보관 > vaildator, reporter job 시작 가능

4) 다른 job에서 꺼내가 사용

vaildator:
    needs: producer
    steps:
        - run: echo "${{ needs.producer.outputs.message }}"
