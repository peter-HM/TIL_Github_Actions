<h2>Concepts of main tools</h2>

Docker (container)
- 음식을 담는 도시락통, 어떤 주방 server에서도 같은 음식이 나오게 함
- app을 환경과 함께 패키징해서 어디서든 똑같이 실행
pros - "내 컴퓨터에서는 안됨" 을 해결, deploy가 빠름
cons - 개념의 난이도, image capicity management need
alter - Podman  root 권한 없이 container 실행, docker와 거의 호환

Github Actions (CI/CD)
- 공장 컨베이어 벨트
- push -> build, test, deploy automatically
pros - github과 통합, free tier enough, setting easy
cons - 난이도 높은 pipe line의 yaml 이 길어질 수 있음
alter - GitLab Cl  자체 서버에 설치 가능, 대기업 선호?

Terraform  (IaC)
- 인프라 설계도면. "EC2 one, RDS one"를 코드로 적으면 AWS 자동 생성
- AWS resouce making, managing, delect using code
pros - 인프라를 코드로 version 관리, aws와 멀티클라우드 지원..
cons - 초반 학습 곡선, 상태파일(tfstate) 관리 주의 필요
alter - AWS CDK  defining infra using python code, only AWS

Kubernetes  (Orchestration)
- Composer of container. 죽으면 살리고, 많아지면 늘린다.
- container delopment, recovery, scaling management automatically
pros - 자동복구, 무중단 배포, 자동 스케일링
cons - 복잡도 높음. app 1~2개면 오버스펙
alter - AWS ECS  K8s보다 쉽고 AWS에 최적화, (1 ~3단계 선택 예정)

Prometheus + Grafana
- Prometheus는 블랙박스 (data collect), Grafana 계기판(Visualization) 둘이 세트로 사용
- server app 매트릭 수집 후 대시보드로 시각화
pros - 업계 표준. 강력한 쿼리 (PromQL) 무료
cons - RAM 사용량 높음
alter - Cloudwatch   AWS 기본 제공. 별도 설치 불필요. (3단계 선택 예정)
