Github Actions

# Best Practice (Performance)

1. Measure less

2. Wait less
- Miniimize runner queuing
- Fail fast

3. Do less
- Conditional filters
- Caching

4. Improve resource usage
- Parallelize (cores + jobs)
- Avoid emulation (g.g. QEMO)

5. Prevent slowdowns


# Best Practices (Caching)

1. Git checkouts

2. Toolchains

3. Dependency downloads

4. Build/Test artifacts

5. Caontainers
- Images
- Layers
- Cache mounts

# Best Practice (Maintainability)

1. Mono vs Multi-repo considerations

2. Define a "CI API" for each service
- use a single task runner or build too! (e.g. Task or Bazel)

3. Avoid duplication by using Composite Actions and or Reuseable Workflows

4. Set up tooling to iterate locally (nektos/act)

# Best Practice (Security)

1. Grant the minimum necessary permissions

2. Avoid long lived credintials where possible

3. Set up allowlist for specific approved actions

4. Pin action versions with the SHA

5. DOn't allow self-hosted runners to execute on fork PRs

6, Require approval to run workflows using environments.
